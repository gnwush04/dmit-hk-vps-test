## 延迟数据怎么看，多少算正常

香港到大陆的物理距离本来就近，正常情况下延迟应该在 **10–30ms** 之间。如果你 Ping 出来超过 60ms，基本可以判断路由绕路了，或者当前节点在高峰期有拥堵。

我自己测的时候，电信到 DMIT 香港的平均延迟在 **15–25ms** 左右，晚高峰偶尔会跳到 35ms，但没有出现过大规模丢包。联通的表现稍微差一点，移动走的是 CMI 线路，延迟比电信略高但也在可接受范围。

几个常用测试命令：

bash
# 基础 Ping 测试
ping 103.135.248.1 -c 100

# 路由追踪（看经过哪些节点）
traceroute 103.135.248.1

# MTR 综合测试（延迟 + 丢包一起看）
mtr -n 103.135.248.1


MTR 是最推荐的工具，能同时看到每一跳的延迟和丢包率，比单纯 Ping 信息量大得多。
